## Abstract

Animals don't move by stringing together independent actions; they fluidly switch between behavioral motifs, each governed by its own dynamics. Premotor and basal ganglia circuits select which behavior to initiate and set its initial conditions, while the primary motor cortex evolves that behavior forward as a dynamical system. Yet computational approaches to modeling naturalistic behavior remain fragmented. While probabilistic state-space models (SSMs) can segment discrete behavioral syllables from keypoint data and reinforcement learning (RL) drives lower-level physics control, there exist no principled bridge between them. We introduce **Code2Act**, a **Switching Nonlinear Dynamical System (S-nLDS)** that unifies these levels within a single imitation framework. Built on observables from [MIMIC-MJX](https://arxiv.org/abs/2511.20532) (Zhang & Yang et al. 2025), a biomechanical simulation of naturalistic rodent behavior, our model replaces the linear regime dynamics of classical switching linear dynamical systems with a RNN trained in RL environment, capable of capturing the nonlinear, non-Gaussian structure of real motor control. The result is a system that jointly segments continuous behavior into discrete semantic units and generates long, biomechanically realistic motor sequences, closing the loop between behavioral understanding and motor generation in a way that mirrors the hierarchical dynamical organization of biological motor control.

</HighlightedSection>

## Problem Statement

Probabilistic state space models (SSMs) capture the dynamics of behavioral trajectories well, segmenting switching and segmental behaviors into discrete syllables, but they fail to serve as generative models of physically realistic trajectories. Reinforcement learning (RL), on the other hand, is widely used for low-level motor control, yet the learned latent representations of end-to-end imitation models are typically unstructured and not reused for regeneration. Current approaches capture what behavior *looks like*, but none can generate it with discrete syllable-level control.

## Pipeline Overview

Code2Act uses a **prior distillation** pipeline where a pre-trained encoder-decoder VAE provides the action generation backbone, and a GRU-based RNN learns to predict the encoder's latent distribution from discrete syllable codes.

## Results

### Training Performance and Reward Decomposition

Code2Act is not designed for trajectory tracking. Given the same KPMS code sequence, the real trajectory could unfold in thousands of different ways. Instead, tracking reward serves as a proxy for learning the **internal dynamics** of each behavioral syllable. Code2Act achieves competitive imitation reward with the oracle VAE (Mimic-MJX) that has access to the full reference trajectory, indicating that the discrete code interface retains enough information to recover the underlying motor programs.

### Distribution Quality (KID)

Tracking reward evaluates whether the decoder follows a *specific* reference trajectory, but the real question is whether Code2Act produces movements that are **distributionally correct** — does the overall repertoire of generated behavior look like real mouse motion? To answer this, we compute Kernel Inception Distance (KID): a VAE is trained on real mocap qpos trajectories to learn a feature space, and then KID measures the distributional gap between real and generated features using an unbiased MMD estimator. Lower KID means the generated motion distribution is closer to real.

<Figure>
  <Picture slot="figure" src={kidBarplot} alt="KID comparison across methods" />
  <Fragment slot="caption">
    **KID across conditions.** Split baseline (gray) is the noise floor from splitting real data. Mimic-MJX (oracle VAE with full trajectory access) and Code2Act (decoder with real KPMS codes) both produce motion close to real, while ARHMM-generated code sequences introduce additional distributional shift. All conditions are evaluated on 200-frame clips; error bars = std across 3 VAE seeds.
  </Fragment>
</Figure>

This metric complements per-trajectory reward: while reward measures how well the decoder tracks individual clips, KID evaluates whether the *distribution* of movements — the range of speeds, postures, and joint configurations — matches real animal behavior. Notably, Code2Act with real codes (KID=1.84) achieves distribution quality comparable to the oracle Mimic-MJX (KID=1.81), suggesting that the discrete code interface preserves the statistical structure of natural behavior.

### Single Codes vs. Code Sequences

A single discrete code alone does not carry behavioral meaning, as individual codes are only behavior primitives, not the actual behaviors we want to describe. It is the **sequence** of codes that gives rise to coherent, identifiable behavior, reflecting how biological motor programs emerge from temporal patterns of neural activity rather than isolated commands.

<Video src="single_code_categories_low.mp4" />

**Single code rollouts.** Repeating a single KPMS code produces only primitive motion fragments. Behavioral identity emerges only when codes are composed into sequences, demonstrating that Code2Act's motor programs are fundamentally **sequential** rather than instantaneous.

### Generative Code Sequences

Code2Act turns behavior generation into a **code sequencing** problem. We fit generative models (empirical transition matrix, HMM, ARHMM) to the KPMS code sequences and use them to generate novel code sequences. These generated sequences produce coherent free-loop behavior without any reference trajectory.

<Video src="code_generation_triptych.mp4" />

**Free-loop generation.** Novel code sequences sampled from learned generative models (Transition Matrix | HMM | ARHMM) produce coherent whole-body behavior. No reference trajectory is provided; the decoder autonomously translates the generated code sequence into motor commands.

<Video src="code_generation_baselines_triptych.mp4" />

**Baseline comparison.** Code sequences generated from baseline models compared against Code2Act, demonstrating the quality of behavior generation across different generative modeling approaches.

## Conclusion

**Code2Act** preserves the internal dynamics of each discrete syllable, turning naturalistic behavior generation into a problem of **code sequencing** rather than repeated interaction with a physics engine. This provides both an engineering solution for planning naturalistic behavioral sequences without relying on physics, and a generative modeling framework, akin to fitting an ARHMM, for synthesizing whole-body behavior. Code2Act bridges the gap between probabilistic state space models, which capture behavioral structure, and reinforcement learning, which produces motor control, by using discrete syllable codes as the interface between structure and action.
